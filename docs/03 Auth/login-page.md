# login-page

Требования к экрану входа (и аналогично — регистрации, с отличиями). Общий поток — [[auth-flow]], проверки — [[Auth Checklist]].

## Поля

- **Email** (обязательно)
- **Пароль** — в режиме «Пароль»; автозаполнение `current-password`

## Кнопки

- Переключатель режимов: **Пароль** | **Ссылка на почту**
- **Войти** / **Отправить ссылку** в зависимости от режима
- Ссылка **Забыли пароль?** → `/forgot-password`
- Ссылка на **регистрацию** — `/signup` (см. [[auth-flow]])

## Ошибки

- Нет конфигурации Supabase на клиенте — **понятный текст**, не stack trace
- Ошибка от Supabase (сеть, rate limit) — короткое сообщение + повторить
- Query `error` / `details` после неудачного callback — декодированное сообщение

## Ссылки

- На **регистрацию** (`/signup`)
- На **сброс пароля** — только если реализован поток; иначе не вешать «битую» ссылку

## Редирект после входа

- Целевой по умолчанию: **`/dashboard`**
- Поддержка **`next`** из query при редиректе с middleware

## Состояния

- **Loading** — кнопка disabled, текст «Отправляем…»
- **Success** — «Ссылка отправлена на …»
- **Error** — одна строка под формой, контрастный цвет

## UX

- Минимум текста, один основной CTA
- Не перегружать опциями до появления парольного режима
- Тёмная тема согласована с остальным приложением
